# Project Overview

## Introduction

The Inventory Management System is a learning project created to understand how enterprise applications are designed using SAP ABAP Development for HANA concepts.

The system is designed to help an organization manage products, suppliers, inventory, and purchase orders in a structured manner.

This project focuses on understanding SAP architecture, business processes, database design, CDS Views, Open SQL, and OData services.


## Problem Statement

Many organizations manage inventory manually or through disconnected systems.

This can result in:

- Incorrect stock information
- Delayed purchase orders
- Duplicate product records
- Difficulty tracking suppliers

An inventory management system provides a centralized solution to manage inventory efficiently.

## Project Scope

The project includes the following modules:

- Product Management
- Supplier Management
- Inventory Management
- Purchase Order Management
- Reporting

## Target Users

The application can be used by:

- Inventory Manager
- Warehouse Staff
- Purchase Manager
- System Administrator

## Functional Requirements

The system should allow users to:

- Add new products
- Update product information
- Manage supplier records
- Track stock availability
- Create purchase orders
- Generate inventory reports

## Non-Functional Requirements

The system should be:

- Easy to use
- Secure
- Scalable
- Reliable
- Optimized for SAP HANA


## Learning Outcome

By working on this project, I aim to improve my understanding of:

- SAP ABAP on HANA
- CDS Views
- Open SQL
- Enterprise application design
- Business process modeling
